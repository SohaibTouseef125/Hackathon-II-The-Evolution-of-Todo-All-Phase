# UI/UX Design

## Overview

This skill provides comprehensive guidance for UI/UX design processes, from initial research to final implementation. It enables Claude to create user-centered designs that follow industry best practices, accessibility standards, and modern design principles.

### 1. Design Process and Methodology
- User research and persona development
- Information architecture and user flows
- Wireframing and prototyping
- Usability testing and iteration

### 2. Visual Design Principles

### 3. Accessibility and Inclusive Design

### 4. Responsive and Adaptive Design

## Design Process Framework

### Phase 1: Research and Discovery
1. **User Research**
   - Conduct user interviews and surveys
   - Create user personas and journey maps
   - Analyze user needs and pain points
   - Define success metrics

2. **Competitive Analysis**
   - Analyze competitor products
   - Identify design patterns and conventions
   - Find opportunities for differentiation

### Phase 2: Design and Prototyping
1. **Information Architecture**
   - Create site maps and user flows
   - Organize content hierarchically
   - Design navigation structures

2. **Wireframing**
   - Create low-fidelity sketches
   - Focus on layout and functionality
   - Test basic usability concepts

3. **Visual Design**
   - Apply design system principles
   - Create high-fidelity mockups
   - Ensure visual consistency

### Phase 3: Testing and Iteration
1. **Usability Testing**
   - Conduct user testing sessions
   - Gather feedback and insights
   - Document findings and recommendations

2. **Iteration**
   - Refine designs based on feedback
   - Address accessibility concerns
   - Optimize for performance

## Design Principles

### 1. Clarity
- Use clear and concise language
- Maintain visual hierarchy
- Ensure content is scannable
- Provide clear navigation cues

### 2. Consistency
- Follow established design patterns
- Maintain consistent spacing and typography
- Use consistent color schemes
- Apply uniform interaction patterns

### 3. Efficiency
- Minimize cognitive load
- Reduce the number of steps needed to complete tasks
- Provide shortcuts for frequent actions
- Use progressive disclosure appropriately

### 4. Accessibility
- Follow WCAG 2.1 AA standards
- Ensure adequate color contrast (4.5:1 minimum)
- Provide alternative text for images
- Support keyboard navigation

## Visual Design Guidelines

### Typography
- **Primary Font**: Use for headings and prominent text
- **Secondary Font**: Use for body text and descriptions
- **Font Scale**: Maintain proper hierarchy (e.g., 12px, 14px, 16px, 20px, 24px, 32px)
- **Line Height**: Use 1.4-1.6 for body text, 1.2-1.4 for headings
- **Letter Spacing**: Use for emphasis or specific stylistic needs

### Color System
- **Primary Color**: Brand color for key actions and highlights
- **Secondary Color**: Supporting color for additional elements
- **Neutral Colors**: Grayscale palette for backgrounds and text
- **Status Colors**: Success (green), warning (yellow), error (red), info (blue)

### Spacing and Layout
- **Grid System**: Use consistent grid (e.g., 8px grid)
- **Padding and Margins**: Maintain consistent spacing (8px, 16px, 24px, 32px)
- **Container Widths**: Define standard container sizes for different screen sizes
- **Responsive Breakpoints**: Define clear breakpoints for different devices

## Accessibility Standards

### Color and Contrast
- Maintain minimum 4.5:1 contrast ratio for normal text
- Maintain 3:1 contrast ratio for large text (18pt+ or 14pt+ bold)
- Don't rely solely on color to convey information
- Test with color blindness simulators

### Keyboard Navigation
- Ensure all interactive elements are keyboard accessible
- Provide visible focus indicators
- Maintain logical tab order
- Support keyboard shortcuts where appropriate

### Screen Reader Compatibility
- Use semantic HTML elements
- Provide descriptive alt text for images
- Use ARIA labels when necessary
- Ensure proper heading hierarchy (h1, h2, h3, etc.)

## Responsive Design Patterns

### Mobile-First Approach
- Start design with smallest screen size
- Progressively enhance for larger screens
- Prioritize essential content and features
- Optimize touch interactions

### Breakpoint Strategy
- **Mobile**: 320px - 767px
- **Tablet**: 768px - 1023px
- **Desktop**: 1024px+
- Consider additional breakpoints as needed

### Touch Interaction
- Minimum touch target size: 44px x 44px
- Provide adequate spacing between touch targets
- Consider thumb-friendly navigation patterns
- Optimize for one-handed use on mobile

## Design Documentation

### Design Specifications
- Provide detailed measurements and dimensions
- Specify colors with hex codes and contrast ratios
- Document typography styles with font families, sizes, weights
- Include interaction states (default, hover, active, focus)

### Component Library
- Document reusable UI components
- Include usage guidelines and variations
- Specify accessibility attributes
- Provide code examples where applicable

### Handoff Documentation
- Prepare assets for development team
- Include responsive behavior specifications
- Document animation and interaction details
- Provide accessibility implementation notes

## User Research Methods

### Quantitative Research
- Analytics analysis
- A/B testing
- Surveys and questionnaires
- Heatmap analysis

### Qualitative Research
- User interviews
- Usability testing sessions
- Focus groups
- Contextual inquiries

### Creating Personas
- Include demographics and goals
- Document pain points and motivations
- Include behavioral patterns
- Validate with real user data

## Prototyping Best Practices

### Low-Fidelity Prototypes
- Focus on structure and flow
- Use simple shapes and placeholders
- Test core functionality concepts
- Iterate quickly based on feedback

### High-Fidelity Prototypes
- Include visual design elements
- Demonstrate interactions and transitions
- Test with real content
- Validate with usability testing

### Interactive Prototypes
- Simulate actual user flows
- Include micro-interactions
- Test across different devices
- Gather feedback on interactions
